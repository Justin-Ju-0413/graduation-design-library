# AI 生成图片检查清单

对照以下要点逐张检查，确定无误后告诉我。

---

## 3.1.png → Fig 3.1 系统架构

应该展示 E203 SoC 的整体结构，包含 CPU、加速器、存储器、外设。

**检查要点：**

- [ ] **E203 CPU 核**在左侧，是一个大框，内部有 IFU 和 EXU 两个子模块
- [ ] **EXU 内部标注了 NICE 接口**（这是 CPU 和加速器的连接点）
- [ ] **CNN Accelerator** 在中间，蓝色或绿色填充，内部有 "NICE Decoder" 和 "4×4 PE Array"
- [ ] **ITCM 64KB** 和 **DTCM 64KB** 在右侧，标注了地址 0x8000_0000 和 0x9000_0000
- [ ] **UART0** 和 **GPIO** 作为外设出现
- [ ] **CLINT** 和 **PLIC** 作为中断控制器出现
- [ ] **Boot ROM (MROM)** 在底部，标注了地址 0x0000_0000
- [ ] 底部有一条 **AHB Bus Fabric** 横线连接所有模块
- [ ] CPU 和 Accelerator 之间有一条加粗/彩色箭头标注了 **NICE Request / Response**
- [ ] **没有** DDR、Ethernet、SD 卡、USB 等不存在的设备

**常见 AI 错误：**
- ❌ 加了 DDR SDRAM 控制器（论文里没有）
- ❌ NICE 接口画成挂在 AHB 总线上（实际是直接连 EXU）
- ❌ 把 PE Array 画成 8×8 或 16×16（实际是 4×4）
- ❌ 没有标注具体地址

## 3.2.png → Fig 3.3 PE 微架构

应该展示单个处理单元（PE）的内部数据通路。

**检查要点：**

- [ ] **输入**：W (INT8) 从上方进入，D (INT8) 从左方进入
- [ ] **乘法器**：标注了 "INT8 × INT8" 或 "8b × 8b"，不是 FP32
- [ ] **累加器**：标注了 "INT32" 或 "32-bit Accumulator"
- [ ] **反馈回路**：累加器输出有一条线绕回去，标注了 "accumulate"
- [ ] **acc_clr 信号**：清零控制信号
- [ ] **en 信号**：使能/时钟门控
- [ ] **ReLU**：可选的非线性激活（负值截断为 0）
- [ ] **输出**：Result (INT32)

**常见 AI 错误：**
- ❌ 用了 FP32 或 float（论文是 INT8 定点）
- ❌ 画成 32×32 乘法器（实际是 8×8）
- ❌ 没有反馈回路（累加器必须带反馈才是 accumulate）
- ❌ 画了一堆多余的流水线寄存器

---

## 3.3.png → Fig 3.4 PE 阵列

应该展示 4×4 的脉动阵列网格结构。

**检查要点：**

- [ ] **4 列 × 4 行**的 PE 方块（不能多不能少）
- [ ] 每个 PE 方块标注了 "PE"
- [ ] **权重 W[0]-W[3]** 从上方流入（纵向箭头）
- [ ] **激活 D[0]-D[3]** 从左方流入（横向箭头）
- [ ] 有虚线或箭头显示数据在行列间的流动方向
- [ ] 底部有 **树形加法器 (Tree Adder)** 汇聚 16 个 PE 结果
- [ ] 最终输出标注了 **Result (INT32)**
- [ ] 标注了 "Output-Stationary" 数据流类型

**常见 AI 错误：**
- ❌ 画成 8×8 或 3×3（论文是 4×4）
- [ ] 权重和激活的流向搞反（W 应该纵向，D 应该横向）
- [ ] 没有树形加法器

---

## 3.4.png → Fig 3.6 FPGA 构建流水线

应该展示从源码到 FPGA 板级验证的完整流程。

**检查要点：**

- [ ] **左轨（软件）**：C Source → ELF Binary → Verilog Hex
- [ ] **右轨（硬件）**：RTL Source → Gate Netlist → Routed Design
- [ ] 两轨标注了工具名（GCC Toolchain / Vivado）
- [ ] 两轨汇合到一个 **Bitstream (.bit)** 块
- [ ] Bitstream 旁边有 **Debug Probes (.ltx)**
- [ ] 最终指向 **Davinci Pro A7-100T FPGA Board**
- [ ] 有 JTAG Programming、ILA Capture、UART Output 标注
- [ ] 有一条 Debug Loop 反馈箭头从底部回到顶部

**常见 AI 错误：**
- ❌ 左右轨搞混（软件在右、硬件在左）
- ❌ Verilog Hex 直接变成 Bitstream（中间缺了硬件轨）
- ❌ 没有 Debug Loop 反馈

---

## 3.5.png → Fig 3.7 验证流程

应该展示多阶段验证策略。

**检查要点：**

- [ ] 四个阶段从左到右排列
- [ ] **Stage 1: RTL Simulation**（Icarus Verilog, RSTAT=19）
- [ ] **Stage 2: Full SoC Simulation**（E203+NICE, PC Trace）
- [ ] **Stage 3: FPGA Bitstream**（Vivado, Timing Closure）
- [ ] **Stage 4: Board Validation**（ILA + UART, LeNet-5 Demo）
- [ ] 阶段之间用箭头连接
- [ ] 有一条红色的 **Debug Loop** 反馈箭头从 Stage 4 回到 Stage 1
- [ ] 标注了 "Root Cause Analysis" 在反馈箭头上

**常见 AI 错误：**
- ❌ 阶段顺序搞反
- ❌ 缺少反馈箭头

## 推荐的开源框图工具

| 工具 | 适合 | 特点 |
|------|------|------|
| **draw.io** | 架构图首选 | 免费，网页/桌面版，导出 SVG/PNG，有大量模板 |
| **Mermaid** | 流程图 | 纯文本描述自动生成图，适合流水线 |
| **Inkscape** | 精细矢量图 | 功能强大但学习成本高 |
| **yEd** | 流程图/网络图 | 自动布局，免费 |

如果你对 AI 生成的图不满意，可以用 draw.io 手动调整后导出替换。
